## ヒーローセクション：2カラム化

### 変更内容
- `.hero-inner` を `displaygrid; grid-template-columns 1fr 1fr; gap24px;` に変更
- 左カラム（`.hero-text`）：既存のh1・p・ボタン群をそのまま内包
- 右カラム（`.hero-media`）：`img src=imageshero-engineer.jpg` を新規配置（写真ファイルは別途配置予定）
- 860px以下（既存のブレークポイントに合わせる）で1カラムに戻し、`.hero-media` に `order-1` を指定して画像を先頭表示

### 判断根拠
- gap値の24pxは、既存CSS内の他の2カラム構成要素（440行台・540行台の2カラムセクション）で使われている値に合わせた
- ブレークポイント860pxは、既存CSSで採用されている値（961行）をそのまま踏襲し、新しい基準を増やさなかった
- モバイル時に画像を上に出す（order-1）理由：ファーストビュー3秒離脱対策という当初の目的上、テキストより先に写真が目に入る方が訴求力が高いと判断

## 「導入の流れ」セクション：フロー矢印追加

### 配置理由
LP内の全セクションのうち、時系列・順序性を持つのは「導入の流れ」のみ（お問い合わせ→ヒアリング→運用確認→初期設定→テスト運用→本番開始、という一方向のプロセス）。他セクション（サービス内容・導入メリット・他社との違い等）は特徴の並列列挙であり、順序性がないためフロー図と本質的に相性が悪い。よってこのセクションを矢印追加の対象とした。

### 変更内容
- 既存の `.steps-grid`  `.step-item`（01〜06の番号付きグリッド）の構造は変更せず維持
- 各 `.step-item`（01〜05）の右端にインラインSVG矢印（`→`）を `positionabsolute` で重ねて配置
- 06（最終ステップ）には矢印を付けない
- 矢印の色は `var(--color-gold)`、背景は `var(--color-bg)` の円形バッジとし、グリッドの罫線と重なっても視認できるようにした

### インラインSVGを採用した理由
- 外部画像ファイル（.svg.png）を別途用意する必要がなく、HTTPリクエストが増えない
- `stroke=currentColor` によりCSS側の `color` プロパティで色を制御できるため、ブランドカラー（ゴールド）との統一が容易

### 既知の制約（許容済み）
- `.steps-grid` は `grid-template-columns repeat(auto-fit, minmax(140px,1fr))` のため、画面幅によっては1行に収まる列数が変動し、行が折り返るケースがある（例：3列×2行など）。この場合、矢印は常に「右隣」を指すため、行末の要素の矢印が視覚的に不自然になる可能性がある。この制約は許容する方針として確認済み。
- 860px以下のモバイル表示（2カラムグリッド）では矢印を非表示にしている。理由：2カラム表示では「右隣」が次のステップではなく別行の要素になるため、矢印が誤った関係性を示してしまうことを避けるため。

## 「解決策」「サービス内容」「導入メリット」3セクションの統合

### 変更内容
- 従来の独立した3セクション（解決策＝説明ボックス／サービス内容＝7ステップのフロー／導入メリット＝4カードのグリッド）を、1つの `.section` に統合した。
- 見出しは統合後に `SESバックオフィス運営のサービス内容`（`section-title`, h2）へ一本化。旧2見出し（「SESバックオフィス運営サービスとは」「サービス内容」「導入メリット」）は廃止。
- セクション内の並び順：導入文（`.solution-box`）→ メリット1〜4（`.service-merits`）→ サービスの流れ（`.service-flow-title` サブ見出し＋既存 `.flow`）。
- 導入文（説明ボックスの3行）と7ステップの各項目名・説明文は変更していない。

### メリットブロックの構成
- 各メリットは `.merit` カード（見出し `h3` ＋ 課題引用 `.merit-quotes` ＋ 解決文 `.merit-solution`）で構成。
- 課題引用は課題提示セクションの文言を『』で囲み、`.merit-quotes p` により1課題=1段落として改行区切りで縦に並べる（`<br>` ではなく `<p>` を採用。1課題を1要素として扱え、行間・余白を CSS で一貫制御できるため）。
- メリット4のみ、対応する課題が課題提示セクションに存在しないため課題引用ブロックを持たない（意図的に引用なし。埋め合わせ文言は追加しない）。
- 引用（課題）は `--color-text-muted`、解決文（地の文）は `--color-text`（白）で色分けし、「お悩み → 解決」の視線移動を明確化した。

### レイアウト判断
- `.service-merits` は `grid-template-columns repeat(2, 1fr)` の2×2グリッド、gap は既存規約の24pxに合わせた。860px（既存ブレークポイント）以下で1カラムに折り返す。
- `.merit` カードの背景・境界・ホバー挙動（`--color-surface` / `--color-border` / ホバーで `--color-gold`）は、旧 `.merit-card` の見た目を踏襲し配色の一貫性を維持した。
- 統合セクションの背景は既定の `--color-bg`（#111111）とし、旧「サービス内容」セクションが持っていた `--color-surface` 背景（`.service` ルール）は不要になったため削除した。
- サービスの流れは既存の丸番号（①②③…相当の `.flow-num`）と縦つなぎ線をそのまま維持。アイコン化は行わず、見た目に問題があれば後日検討とする。

### 除去したもの
- 旧 `.service`（フロー用サーフェス背景）ルール。
- 旧 `.merits-grid` / `.merit-card` / `.merit-icon` 系ルールと、各メリットカードのインラインSVGアイコン（新構成ではアイコンを持たないため）。
