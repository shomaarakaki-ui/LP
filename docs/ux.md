# LP Section UX Guidelines

このドキュメントは、CoreDesk LPのセクション設計を再現するための固定ルールです。特定セクションを基準名として参照せず、余白・文字量・カード密度・CTA配置の考え方を言語化して残します。

## 基本方針

- 1セクションは「見出し」「短い説明」「主情報」「必要なCTA」の順に組み立てる。
- 説明で説得しすぎず、見出し・カード・図解・箇条書きで理解できる状態にする。
- 装飾は情報の理解を助けるものだけに絞る。背景の大きな薄文字や意味の薄い飾りは原則使わない。
- PCでは、主要セクションが1画面内に収まりやすい縦密度を目指す。
- スマホでは、横並びを無理に維持せず、縦方向の流れを明確にする。

## セクションの密度

- 標準的な上下余白は `80px` 前後を目安にする。
- 情報量が少ないセクションは `64px` まで圧縮してよい。
- ヒーローやクロージングなど、画面として見せる役割の強いセクションのみ大きめの高さを許容する。
- ファーストビューは画面設計そのものが崩れやすいため、セクション高さを不用意に削らない。縦幅を整える場合は、高さではなく内部の余白・行間・CTA前余白から調整する。
- 固定ヘッダーがあるため、PCの1画面設計はヘッダー分を差し引いて考える。アンカー補正やファーストビュー余白はヘッダー高さのCSS変数を使い、1画面セクションの見せ方は下記の基準値を使う。
- 1画面セクションとして見せたい場合は、PCで `min-height: calc(100vh - 60px)` を使い、内容を上下中央に配置する。スマホでは固定高さにせず、自然な縦スクロールに戻す。
- 主情報までの余白は `40px` から `48px` 程度を目安にする。
- CTAまでの余白は `36px` から `48px` 程度に抑える。
- セクションの区切りは下線を基本にする。上下両方に線を入れると硬く見えやすいため、ページの流れを保ちながら下線で次の区画を示す。
- セクション背景は白系の基本背景Aと薄い面背景Bを交互に使い、区切りを視覚的に補助する。
- 強いCTA背景が必要な場合でも黒は避ける。基本背景Bを少しだけ濃くした背景Cを使い、主張を抑えながら通常セクションとの差をつける。

## 文字量

- セクション説明文は原則1から2文にする。
- カード内の説明は1文を基本にする。
- 長い説明が必要な場合も、見出しで要点を立て、本文は補足にする。
- CTAの本文は1文だけにする。CTAは読ませる場所ではなく、次の行動を選ばせる場所とする。
- ファーストビューの本文は、ゆったり見せたい場合でも行間を過度に広げすぎない。見出し下余白、本文行間、本文下余白の順に微調整する。

## カード設計

- カードは見出し、短い本文、必要ならアイコンや画像だけで構成する。
- カード内余白は `22px` から `32px` 程度を目安にする。
- カード間のgapはPCで `20px` から `24px` 程度を基本にする。
- カードの角丸は控えめにする。大きな丸みはLP全体の実務感を弱めるため避ける。
- カード内の本文行間は `1.7` から `1.85` 程度に抑える。
- 料金や比較など情報の階層が明確なカードは、構造を変えずに、グリッド上余白・カード内余白・区切り線余白・リストgapだけを軽く調整する。
