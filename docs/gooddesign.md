# planセクションのいいところ

- セクション見出しを大きくしすぎず、カードを主役にしている。
- 見出しエリアからカードグリッドまで `40px` 程度の余白を取り、情報の切り替わりが分かりやすい。
- カード内でプラン名、価格、機能リストの文字サイズに差をつけ、視線の順番を作っている。
- カード内に区切り線を入れ、主要情報と補足情報を分けている。
- カードの余白、gap、角丸が控えめで、実務向けLPとして落ち着いて見える。
- CTAをセクション本体から分け、プラン本体を比較と理解に集中させている。
- CTAは短い本文と明確なボタンに絞り、主情報より目立たせすぎていない。

## 余白の観点で見るplanセクション

- セクション見出しとカードグリッドの間に `40px` の余白があり、「説明を読む領域」と「比較する領域」が分かれている。
- カード同士の gap は `24px`。カードを近づけすぎず、各プランを独立した選択肢として見せている。
- カード内の padding は `32px 30px`。価格や機能リストを詰め込まず、プランごとの情報を落ち着いて読める。
- プラン名と価格の間は `8px`、価格下は `4px` と近く、同じ主要情報のまとまりとして見える。
- 価格と機能リストの間には区切り線があり、線の上下に `20px` の余白を取っている。これにより「価格」と「含まれる内容」が混ざらない。
- 機能リストは `gap: 8px` で、項目を詰めすぎず、ただしカードが間延びしすぎない密度にしている。
- プラン本体とCTAの間は `48px` 空け、さらにCTA内の上部に `24px` の余白と罫線を入れている。カード比較が終わった後に、別行動として相談導線へ移れる。
- おすすめカードは `translateY(-12px)` で少しだけ持ち上げているが、余白とカード間隔があるため、他カードとぶつからず自然に目立つ。

## 他セクションに転用する時の注意

- 情報を混ぜる場合でも、主役にしたい要素の前後には `40px` 前後の余白を置く。
- 補足情報はカード化しすぎず、線・小さな文字・控えめな余白で「補足」に見せる。
- CTAはカード群の直後に詰めず、`48px` 前後の間を空けて、理解フェーズと行動フェーズを分ける。
- カード内は、見出し・主要情報・補足情報のまとまりごとに余白を変える。同じまとまりは近く、違うまとまりは離す。
- 余白を削って情報を1画面に収めるより、情報の主従が分かる余白を優先する。

## 「依頼できる業務内容」セクションに実際に転用した具体ポイント

planセクションを参照して、tasksセクションの視認性を上げるために転用した具体値・考え方。

### 色使い（金色を主張させない）
- カード見出しの「金色ベタ塗りバー＋白文字」をやめ、planと同じく **金色は文字色・細い罫線・箇条書きマーカーだけ**のアクセントに絞った。面で金色を使わない。
- 本文（詳細テキスト）は `--color-text`（濃い黒）ではなく **`--color-text-muted`（#6B6B6B）** に。planの機能リストと同じ扱いで、見出しとの主従が生まれ、文字が「重い・でかい」印象が消える。
- 箇条書きマーカーは planの `—`（金色）に統一。ドットより控えめで、実務LPらしく落ち着く。

### 文字サイズの主従
- planの「プラン名 > 価格 > 機能リスト」のように差をつける発想を流用。
  - カード見出し `15px` 太字（金色）／詳細 `14px`・ミュート色。
- 詰め込まず、サイズ差＋色差の2軸で視線の順番を作る（planの機能リスト `14px`・ミュート色をそのまま踏襲）。

### 余白・区切り・角丸
- カード背景は **`--color-surface`**、角丸は **`2px`**（planカードと統一。8pxの丸みをやめて落ち着かせた）。
- カード内 padding は planの `32px 30px` を目安に、情報量に合わせ `26px 28px`（モバイル `22px`）へ調整。
- 見出しと詳細の間に **区切り線＋前後 `14px` 余白**（planの `plan-divider` の考え方）。主要情報と補足を分ける。
- 詳細リストは planと同じ **`gap: 8px`**。詰めすぎず間延びさせない密度。

### セクション構成
- 見出しエリアからコンテンツまで planと同じく **`40px` 前後の余白**を確保（`scope-flow-block` の `margin-top: 40px`）。
- 目障りな装飾（背景の大きな英字ウォーターマーク）は削除。planセクションが装飾に頼らず情報の主従で成立していることを基準にした。

### 補足（この考え方が効く理由）
- 「シンプルにする＝視認性が上がる」を、planは **面で色を使わず・サイズと色の差で主従をつくる**という形で実現している。tasksセクションもこれに寄せることで、装飾を足さずに読みやすくできた。
